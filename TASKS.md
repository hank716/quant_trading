## 🤖 自動運作協議（每次啟動先看這段）

### 啟動流程（每次必做）

```bash
# 1. 確認當前狀態
git status
git branch --show-current
git log --oneline -10

# 2. 同步遠端
git fetch origin
git pull origin develop  # 如果在 develop 上

# 3. 檢查 BLOCKED.md 是否有待處理事項
cat BLOCKED.md 2>/dev/null || echo "No blockers"

# 4. 找到 TASKS.md 第一個 [ ] 未完成任務，繼續做
```

### Git Workflow 規則（非常重要）

**分支策略：**
- `main` — 保護分支，**你絕對不能直接 push 或 merge**。只有人工審核後才會合併
- `develop` — 整合分支，**你可以 merge feature branch 進來**
- `feat/phase{N}-{description}` — 你工作的地方

**每個 Phase 的標準流程：**
```bash
# 1. 從最新的 develop 開 feature branch
git checkout develop
git pull origin develop
git checkout -b feat/phaseN-description

# 2. 做子任務，每完成一個就 commit
git add <files>
git commit -m "feat(phaseN): <具體描述>"

# 3. 跑完整測試
pytest -q
# 如果失敗，修到通過再繼續

# 4. 推到遠端
git push -u origin feat/phaseN-description

# 5. Phase 驗收通過後，建立 PR 並自動 merge 回 develop
gh pr create --base develop --head feat/phaseN-description \
  --title "Phase N: <標題>" \
  --body "$(cat <<'EOF'
## 做了什麼
<列出主要改動>

## 測試結果
- pytest: PASS
- docker build: PASS
- <其他驗證>

## 待確認事項（若有）
<列出需要人工 review 的決策>

🤖 Auto-generated by Claude Code
EOF
)"

# 6. 自動 merge 到 develop（僅限 feature → develop，不可 → main）
gh pr merge --squash --auto --delete-branch
```

### Commit Message 規範

使用 Conventional Commits：
- `feat(phaseN):` — 新功能
- `fix(phaseN):` — 修 bug
- `test(phaseN):` — 新增/修改測試
- `docs(phaseN):` — 文件
- `refactor(phaseN):` — 重構
- `chore(phaseN):` — 雜項（依賴更新、配置）

範例：
```
feat(phase0): add docker-compose skeleton with quant-ui service

- Created compose/docker-compose.yml with quant-ui, prometheus, grafana
- Added bind mount for workspace/
- UI smoke test passes
```

### 完成子任務的定義（Definition of Done）

一個子任務 `[ ]` 可以改成 `[x]` 的條件**全部**要滿足：
1. 程式碼寫完且符合原檔案風格
2. 對應的 `tests/unit/` 測試寫完
3. `pytest -q` 全過（包含 `test/test_decision_system.py`）
4. 已 commit（commit message 符合規範）
5. 如果是新模組，有 docstring 說明用途

### 遇到卡點的處理

**什麼情況算卡點：**
- 需要真實 API credentials（pCloud、Supabase、FinMind）
- 架構決策不明確（多種合理做法）
- 外部服務呼叫失敗且無法 mock
- 現有測試有 bug 且不確定是不是要修

**卡點處理步驟：**
1. 在根目錄建立或更新 `BLOCKED.md`
2. 寫清楚卡在哪、需要什麼、暫時怎麼處理
3. **跳過這個子任務**，繼續做下一個能做的
4. 如果整個 Phase 都卡住，開 PR 說明狀態，等待人工介入

### 禁止事項（絕對不能做）

- ❌ 直接 push 到 `main`
- ❌ `git push --force`（除非明確被指示）
- ❌ 把 API key、token、password 寫進 commit
- ❌ 修改 `.git/hooks/` 或 `.github/workflows/` 裡面已存在的保護機制
- ❌ 跳過測試（`pytest -x` OK，但不可 `--ignore`）

  ```python
  """pCloud API client - Phase 0 mock skeleton"""
  import os
  import logging
  from pathlib import Path
  from typing import Optional

  logger = logging.getLogger(__name__)


  class PCloudClient:
      """pCloud API wrapper. 無 token 時自動進 mock mode."""

      def __init__(self, token: Optional[str] = None, region: str = "eu"):
          self.token = token or os.getenv("PCLOUD_TOKEN")
          self.region = region or os.getenv("PCLOUD_REGION", "eu")
          self.mock_mode = not self.token
          self.base_url = (
              "https://eapi.pcloud.com" if self.region == "eu"
              else "https://api.pcloud.com"
          )
          if self.mock_mode:
              logger.warning("PCloudClient: running in MOCK mode (no token)")

      def mkdir(self, remote_path: str) -> dict:
          if self.mock_mode:
              logger.info(f"[MOCK] mkdir {remote_path}")
              return {"mock": True, "path": remote_path}
          raise NotImplementedError("Real API impl in Phase 2")

      def upload_file(self, local_path: Path, remote_path: str) -> dict:
          if self.mock_mode:
              logger.info(f"[MOCK] upload {local_path} -> {remote_path}")
              return {"mock": True, "local": str(local_path), "remote": remote_path}
          raise NotImplementedError("Real API impl in Phase 2")

      def download_file(self, remote_path: str, local_path: Path) -> dict:
          if self.mock_mode:
              logger.info(f"[MOCK] download {remote_path} -> {local_path}")
              return {"mock": True, "remote": remote_path, "local": str(local_path)}
          raise NotImplementedError("Real API impl in Phase 2")

      def list_folder(self, remote_path: str) -> list:
          if self.mock_mode:
              logger.info(f"[MOCK] list {remote_path}")
              return []
          raise NotImplementedError("Real API impl in Phase 2")

      def file_checksum(self, remote_path: str) -> Optional[str]:
          if self.mock_mode:
              return None
          raise NotImplementedError("Real API impl in Phase 2")
  ```
